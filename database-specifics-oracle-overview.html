Oracle databases can be accessed by the .NET ODP driver from Oracle. Telerik OpenAccess ORM refers to the ODP driver version 2.102.2.20. The drivers can be downloaded from here. The minimal version should be 10.2.0.2.20.

For Oracle, you will need an account on an active database. Telerik OpenAccess ORM will automatically create/update the class schema information.

Connect with your RDBMS administrator if you need help.

Please refer to the configuration files, which are present in your Telerik OpenAccess ORM installation for more information on how to configure the various backends.

Since Telerik OpenAccess ORM takes care of the communication and operations necessary to store and retrieve your objects, you don’t need to have experience with the particular relational system or knowledge of SQL.

By using the advanced features of Telerik OpenAccess ORM, you have a full control over the Telerik OpenAccess ORM O/R mapper. But this "fine tuning" is not needed to implement sophisticated applications using Telerik OpenAccess ORM.
Telerik OpenAccess ORM has been tested with the 9, 10R2 and 11g.
